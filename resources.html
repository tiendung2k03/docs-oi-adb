Câu hỏi thường gặp (FAQ)

OIADB là gì?

OIADB là một thư viện Python đóng gói (wrapper) cho Android Debug Bridge (ADB). Nó cung cấp một giao diện Pythonic để tương tác với các thiết bị Android, bao gồm các tính năng nâng cao như nhận diện hình ảnh và tương tác qua XML Dump (API trợ năng).

OIADB có miễn phí không?

Có, OIADB được phát hành dưới giấy phép MIT, cho phép sử dụng miễn phí cho cả mục đích cá nhân và thương mại.

Tôi cần cài đặt gì để sử dụng OIADB?

Bạn cần cài đặt Python (phiên bản 3.7 trở lên). OIADB sẽ tự động tải xuống và cài đặt ADB nếu cần thiết. Bạn cũng cần bật chế độ USB Debugging trên thiết bị Android của mình.

Để sử dụng tính năng nhận diện hình ảnh, bạn cần cài đặt OpenCV:

pip install opencv-python

Nhận diện hình ảnh hoạt động như thế nào?

OIADB chụp ảnh màn hình thiết bị và sử dụng thuật toán so khớp mẫu của OpenCV để tìm kiếm hình ảnh mẫu bạn cung cấp. Nó trả về tọa độ và độ tương đồng nếu tìm thấy.

Làm thế nào để cải thiện độ chính xác của nhận diện hình ảnh?

Sử dụng hình ảnh mẫu rõ ràng, cắt chính xác, có độ tương phản tốt. Điều chỉnh ngưỡng tương đồng (threshold). Thử sử dụng tìm kiếm đa tỷ lệ, góc xoay hoặc phát hiện cạnh Canny nếu cần.

XML Dump là gì?

XML Dump là quá trình truy xuất cấu trúc giao diện người dùng (UI) của ứng dụng Android dưới dạng file XML. File này chứa thông tin về tất cả các phần tử trên màn hình, cho phép bạn tương tác với chúng thông qua API trợ năng.

Khi nào nên sử dụng XML Dump thay vì nhận diện hình ảnh?

XML Dump thường ổn định và nhanh hơn nhận diện hình ảnh, đặc biệt khi UI không thay đổi thường xuyên. Nó phù hợp để điền form, điều hướng ứng dụng và trích xuất dữ liệu văn bản. Nhận diện hình ảnh phù hợp hơn cho các phần tử đồ họa, game hoặc khi cấu trúc UI thay đổi động.

Thiết bị của tôi không được nhận diện?

Đảm bảo bạn đã bật USB Debugging trên thiết bị và cấp quyền cho máy tính. Kiểm tra cáp USB và cổng kết nối. Thử chạy lệnh adb devices trong terminal để xem thiết bị có được liệt kê không.

Làm thế nào để báo cáo lỗi hoặc yêu cầu tính năng mới?

Bạn có thể tạo một issue trên trang GitHub của dự án. Vui lòng cung cấp mô tả chi tiết về lỗi hoặc tính năng bạn mong muốn.

Xử lý sự cố

Dưới đây là một số vấn đề phổ biến và cách khắc phục khi sử dụng OIADB.

Lỗi kết nối thiết bị

Thiết bị không được liệt kê (adb devices)

  • Kiểm tra xem USB Debugging đã được bật trên thiết bị chưa (Settings > Developer options).
  • Kiểm tra cáp USB và cổng kết nối.
  • Cấp quyền gỡ lỗi USB cho máy tính trên thiết bị (thường có thông báo pop-up khi kết nối lần đầu).
  • Cài đặt driver ADB phù hợp cho thiết bị của bạn trên máy tính.
  • Thử khởi động lại thiết bị và máy tính.
  • Chạy adb kill-server và adb start-server trong terminal.

Thiết bị hiển thị là "unauthorized"

  • Rút cáp USB và cắm lại.
  • Trên thiết bị, thu hồi quyền gỡ lỗi USB (Revoke USB debugging authorizations) trong Developer options và cấp lại quyền.
  • Xóa file adbkey và adbkey.pub trong thư mục .android của người dùng trên máy tính và thử kết nối lại.

Lỗi nhận diện hình ảnh

Không tìm thấy hình ảnh (find_image trả về None)

  • Giảm ngưỡng tương đồng (threshold).
  • Đảm bảo hình ảnh mẫu được chụp từ cùng độ phân giải và không bị thay đổi tỷ lệ.
  • Kiểm tra xem phần tử có thực sự hiển thị trên màn hình không.
  • Thử sử dụng tìm kiếm đa tỷ lệ (scale_range) nếu kích thước có thể thay đổi.
  • Thử sử dụng phát hiện cạnh Canny (use_canny=True) nếu hình ảnh mờ.
  • Chụp lại hình ảnh mẫu chính xác hơn.

Tìm thấy hình ảnh sai vị trí

  • Tăng ngưỡng tương đồng (threshold).
  • Sử dụng hình ảnh mẫu có đặc trưng độc đáo hơn.
  • Giới hạn vùng tìm kiếm (region) nếu có thể.

Lỗi XML Dump

Lỗi khi lấy XML (get_ui_xml)

  • Đảm bảo thiết bị đã kết nối và được cấp quyền.
  • Kiểm tra xem uiautomator có hoạt động trên thiết bị không (thử chạy adb shell uiautomator dump /sdcard/ui.xml).
  • Thử khởi động lại thiết bị.
  • Đảm bảo không có ứng dụng nào khác đang sử dụng uiautomator.

Không tìm thấy phần tử (find_element_by_...)

  • Kiểm tra lại văn bản hoặc thuộc tính tìm kiếm (chính tả, phân biệt hoa thường).
  • Đảm bảo phần tử thực sự tồn tại trong cấu trúc XML (kiểm tra file XML được dump ra).
  • Lấy lại cấu trúc XML mới nhất trước khi tìm kiếm nếu UI đã thay đổi.
  • Thử tìm kiếm với partial_match=True hoặc case_sensitive=False.

Vấn đề hiệu suất

Nhận diện hình ảnh quá chậm

  • Giới hạn vùng tìm kiếm (region).
  • Giảm kích thước hình ảnh mẫu.
  • Tránh sử dụng tìm kiếm đa tỷ lệ và góc xoay nếu không cần thiết.
  • Đảm bảo use_gray=True.

Lấy XML Dump quá chậm

  • Sử dụng XML Server (start_xml_server và get_ui_xml_from_server) nếu cần lấy XML nhiều lần.
  • Lưu trữ và tái sử dụng cấu trúc XML khi UI không thay đổi.

Báo cáo lỗi

Nếu bạn gặp phải lỗi không thể khắc phục, vui lòng báo cáo trên GitHub Issues. Khi báo cáo lỗi, hãy cung cấp các thông tin sau:

  • Phiên bản OIADB bạn đang sử dụng.
  • Phiên bản Python.
  • Hệ điều hành của bạn.
  • Thông tin thiết bị Android (Model, phiên bản Android).
  • Mô tả chi tiết về lỗi và các bước để tái hiện lỗi.
  • Mã nguồn gây ra lỗi (nếu có thể).
  • Thông báo lỗi đầy đủ (traceback).

Đóng góp

Chúng tôi hoan nghênh mọi đóng góp cho dự án OIADB! Dù là báo cáo lỗi, yêu cầu tính năng, cải thiện tài liệu hay đóng góp mã nguồn.

Hướng dẫn đóng góp

Vui lòng tham khảo Hướng dẫn đóng góp trên GitHub để biết chi tiết về quy trình đóng góp, quy tắc mã hóa và cách tạo Pull Request.

Đóng góp mã nguồn

  1. Fork repository oiadb.
  2. Tạo một nhánh mới cho tính năng hoặc sửa lỗi của bạn (git checkout -b feature/AmazingFeature).
  3. Thực hiện các thay đổi và commit (git commit -m 'Add some AmazingFeature').
  4. Đẩy lên nhánh của bạn (git push origin feature/AmazingFeature).
  5. Mở một Pull Request trên GitHub.

Đóng góp tài liệu

Nếu bạn phát hiện lỗi chính tả, thông tin sai lệch hoặc muốn cải thiện tài liệu, bạn có thể:

  • Tạo một issue trên GitHub để mô tả vấn đề.
  • Hoặc, chỉnh sửa trực tiếp file Markdown trong thư mục docs trên GitHub và tạo Pull Request.
