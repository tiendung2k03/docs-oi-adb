Hướng dẫn OIADB

Nhận diện hình ảnh

Cơ bản về nhận diện hình ảnh

Tính năng nhận diện hình ảnh của OIADB cho phép bạn tìm kiếm và tương tác với các phần tử trên màn hình dựa trên hình ảnh mẫu, mà không cần biết tọa độ cố định hoặc cấu trúc UI.

Cách hoạt động

OIADB sử dụng thuật toán so khớp mẫu (template matching) của OpenCV để tìm kiếm hình ảnh mẫu trên ảnh chụp màn hình thiết bị. Quá trình này bao gồm các bước sau:

  1. Chụp ảnh màn hình thiết bị
  2. Tải hình ảnh mẫu từ đường dẫn được cung cấp
  3. Áp dụng thuật toán so khớp mẫu để tìm vị trí tốt nhất
  4. Trả về tọa độ và độ tương đồng nếu vượt qua ngưỡng

Dưới đây là ví dụ cơ bản về cách sử dụng nhận diện hình ảnh:

from oiadb import MyADB
from oiadb.commands import image_interaction

# Khởi tạo ADB
adb = MyADB()

# Tìm hình ảnh trên màn hình
result = image_interaction.find_image("/path/to/button.png", threshold=0.8)

if result:
    x, y, confidence = result
    print(f"Đã tìm thấy hình ảnh tại ({x}, {y}) với độ tương đồng {confidence}")
    
    # Nhấp vào vị trí đã tìm thấy
    image_interaction.tap_image("/path/to/button.png")
else:
    print("Không tìm thấy hình ảnh")

Chuẩn bị hình ảnh mẫu

Chất lượng của hình ảnh mẫu có ảnh hưởng lớn đến độ chính xác của nhận diện. Dưới đây là một số hướng dẫn để tạo hình ảnh mẫu hiệu quả:

Cắt chính xác

Cắt hình ảnh mẫu để chỉ bao gồm phần tử bạn muốn tìm, loại bỏ các phần không cần thiết xung quanh.

Đặc trưng rõ ràng

Chọn hình ảnh có đặc trưng rõ ràng và dễ phân biệt với các phần khác của màn hình.

Kích thước phù hợp

Hình ảnh mẫu không nên quá nhỏ (dưới 20x20 pixel) hoặc quá lớn (trên 1/4 màn hình).

Độ tương phản tốt

Hình ảnh có độ tương phản tốt giữa phần tử và nền sẽ dễ nhận diện hơn.

Lưu ý quan trọng

Hình ảnh mẫu phải được chụp từ cùng một độ phân giải và tỷ lệ màn hình với thiết bị mục tiêu. Nếu bạn thay đổi thiết bị hoặc độ phân giải, bạn có thể cần tạo lại hình ảnh mẫu.

Dưới đây là quy trình được đề xuất để tạo hình ảnh mẫu:

  1. 1

    Chụp ảnh màn hình

    Chụp ảnh màn hình thiết bị mục tiêu với phần tử bạn muốn tìm kiếm.

    from oiadb import MyADB

adb = MyADB()
adb.execute_command("shell screencap -p /sdcard/screenshot.png")
adb.execute_command("pull /sdcard/screenshot.png .")
  2. 2

    Cắt phần tử mục tiêu

    Sử dụng công cụ chỉnh sửa ảnh để cắt chính xác phần tử bạn muốn tìm kiếm.

  3. 3

    Lưu hình ảnh mẫu

    Lưu hình ảnh đã cắt với định dạng PNG để giữ nguyên chất lượng.

  4. 4

    Kiểm tra hình ảnh mẫu

    Kiểm tra xem hình ảnh mẫu có thể được tìm thấy trên màn hình không.

    result = image_interaction.find_image("template.png")
print(f"Kết quả tìm kiếm: {result}")

Kỹ thuật nâng cao

OIADB cung cấp nhiều tùy chọn nâng cao để xử lý các trường hợp phức tạp trong nhận diện hình ảnh.

Tìm kiếm với nhiều tỷ lệ

Khi hình ảnh mẫu có thể xuất hiện với kích thước khác nhau trên màn hình, bạn có thể sử dụng tìm kiếm đa tỷ lệ:

result = image_interaction.find_image(
    "template.png",
    scale_range=(0.5, 1.5),  # Tìm từ 50% đến 150% kích thước gốc
    scale_steps=10            # Số bước tỷ lệ
)

Lưu ý rằng tìm kiếm đa tỷ lệ sẽ chậm hơn tìm kiếm thông thường.

Tìm kiếm với góc xoay

Nếu phần tử có thể xuất hiện với các góc xoay khác nhau:

result = image_interaction.find_image(
    "template.png",
    rotation_range=(-30, 30),  # Tìm từ -30 đến +30 độ
    rotation_steps=6           # Số bước góc xoay
)

Tìm kiếm với góc xoay sẽ làm giảm đáng kể hiệu suất, chỉ sử dụng khi cần thiết.

Sử dụng phát hiện cạnh Canny

Đối với hình ảnh mờ hoặc có độ tương phản thấp, phát hiện cạnh Canny có thể cải thiện kết quả:

result = image_interaction.find_image(
    "template.png",
    use_canny=True,  # Bật phát hiện cạnh Canny
    threshold=0.6    # Có thể sử dụng ngưỡng thấp hơn với Canny
)

Tìm kiếm trong vùng cụ thể

Để cải thiện hiệu suất và độ chính xác, bạn có thể giới hạn tìm kiếm trong một vùng cụ thể:

result = image_interaction.find_image(
    "template.png",
    region=(100, 200, 400, 300)  # (x, y, width, height)
)

Xử lý sự cố

Dưới đây là một số vấn đề thường gặp khi sử dụng nhận diện hình ảnh và cách khắc phục:

Không tìm thấy hình ảnh

Hình ảnh mẫu không được tìm thấy trên màn hình.

Giải pháp

  • Giảm ngưỡng tương đồng (threshold) xuống 0.7 hoặc thấp hơn
  • Kiểm tra xem hình ảnh mẫu có đúng độ phân giải không
  • Thử sử dụng tìm kiếm đa tỷ lệ
  • Chụp lại hình ảnh mẫu từ thiết bị mục tiêu
# Thử với ngưỡng thấp hơn
result = image_interaction.find_image("template.png", threshold=0.7)

Kết quả không chính xác

Hình ảnh được tìm thấy nhưng ở vị trí sai.

Giải pháp

  • Tăng ngưỡng tương đồng lên 0.9 hoặc cao hơn
  • Sử dụng hình ảnh mẫu có đặc trưng rõ ràng hơn
  • Thử sử dụng phát hiện cạnh Canny
# Thử với ngưỡng cao hơn và phát hiện cạnh
result = image_interaction.find_image(
    "template.png", 
    threshold=0.9,
    use_canny=True
)

Hiệu suất chậm

Tìm kiếm hình ảnh mất quá nhiều thời gian.

Giải pháp

  • Giới hạn vùng tìm kiếm bằng tham số region
  • Giảm kích thước hình ảnh mẫu
  • Giảm số bước tỷ lệ và góc xoay
  • Đảm bảo use_gray=True để tăng tốc độ
# Tối ưu hiệu suất
result = image_interaction.find_image(
    "template.png",
    region=(100, 200, 400, 300),  # Giới hạn vùng tìm kiếm
    use_gray=True                  # Sử dụng ảnh xám
)

XML Dump & Trợ năng

Cơ bản về XML Dump

XML Dump là một tính năng mạnh mẽ của OIADB cho phép bạn truy xuất cấu trúc UI của ứng dụng Android dưới dạng XML và tương tác với các phần tử thông qua API trợ năng.

Cách hoạt động

OIADB sử dụng lệnh uiautomator dump của ADB để truy xuất cấu trúc UI hiện tại của thiết bị Android. Cấu trúc này chứa thông tin về tất cả các phần tử UI trên màn hình, bao gồm vị trí, kích thước, văn bản và các thuộc tính khác.

Dưới đây là ví dụ cơ bản về cách sử dụng XML Dump:

from oiadb import MyADB
from oiadb.commands import xml_dump

# Khởi tạo ADB
adb = MyADB()

# Lấy cấu trúc XML của giao diện người dùng hiện tại
xml_tree = xml_dump.get_ui_xml()

# In ra cấu trúc XML đã định dạng
print(xml_dump.format_xml(xml_tree))

# Lưu cấu trúc XML vào file
xml_dump.save_xml(xml_tree, "ui_dump.xml")

Tìm phần tử UI

OIADB cung cấp nhiều phương thức để tìm kiếm các phần tử trong cấu trúc UI dựa trên văn bản, thuộc tính hoặc vị trí.

Tìm phần tử dựa trên văn bản hiển thị:

# Tìm phần tử có văn bản chính xác
element = xml_dump.find_element_by_text("Login")

# Tìm phần tử có văn bản chứa chuỗi con
element = xml_dump.find_element_by_text("Log", partial_match=True)

# Tìm phần tử có văn bản không phân biệt hoa thường
element = xml_dump.find_element_by_text("login", case_sensitive=False)

if element:
    print(f"Đã tìm thấy phần tử: {xml_dump.get_element_info(element)}")
else:
    print("Không tìm thấy phần tử")

Tìm phần tử dựa trên thuộc tính:

# Tìm theo resource-id
element = xml_dump.find_element_by_attribute(
    "resource-id", 
    "com.example.app:id/username"
)

# Tìm theo class
element = xml_dump.find_element_by_attribute(
    "class", 
    "android.widget.Button"
)

# Tìm theo content-desc
element = xml_dump.find_element_by_attribute(
    "content-desc", 
    "Submit button"
)

# Tìm theo nhiều thuộc tính (AND logic)
element = xml_dump.find_element_by_attributes({
    "class": "android.widget.Button",
    "clickable": "true"
})

Tìm phần tử dựa trên vị trí trên màn hình:

# Tìm phần tử tại tọa độ cụ thể
element = xml_dump.find_element_at_position(500, 800)

if element:
    # Lấy thông tin về phần tử
    bounds = xml_dump.get_element_bounds(element)
    print(f"Phần tử có kích thước: {bounds}")
    
    # Kiểm tra xem phần tử có thể nhấp không
    is_clickable = xml_dump.is_element_clickable(element)
    print(f"Có thể nhấp: {is_clickable}")

Tìm nhiều phần tử cùng một lúc:

# Tìm tất cả các nút
buttons = xml_dump.find_elements_by_attribute("class", "android.widget.Button")
print(f"Đã tìm thấy {len(buttons)} nút")

# Tìm tất cả các phần tử có thể nhấp
clickable_elements = xml_dump.find_elements_by_attribute("clickable", "true")
print(f"Đã tìm thấy {len(clickable_elements)} phần tử có thể nhấp")

# Lặp qua các phần tử và in thông tin
for i, element in enumerate(buttons):
    text = xml_dump.get_element_text(element) or "Không có văn bản"
    print(f"Nút {i+1}: {text}")

Tương tác với phần tử

Sau khi tìm thấy phần tử, bạn có thể tương tác với nó thông qua các phương thức của OIADB:

Nhấp vào phần tử

# Tìm và nhấp vào nút đăng nhập
login_button = xml_dump.find_element_by_text("Login")
if login_button:
    xml_dump.click_element(login_button)

Nhập văn bản

# Tìm trường nhập liệu và nhập văn bản
username_field = xml_dump.find_element_by_attribute(
    "resource-id", 
    "com.example.app:id/username"
)
if username_field:
    xml_dump.input_text(username_field, "user@example.com")

Xóa văn bản

# Tìm trường nhập liệu và xóa văn bản
password_field = xml_dump.find_element_by_attribute(
    "resource-id", 
    "com.example.app:id/password"
)
if password_field:
    xml_dump.clear_text(password_field)

Cuộn đến phần tử

# Tìm và cuộn đến phần tử
terms_element = xml_dump.find_element_by_text("Terms of Service")
if terms_element:
    xml_dump.scroll_to_element(terms_element)

Kiểm tra trạng thái phần tử

OIADB cung cấp các phương thức để kiểm tra trạng thái của phần tử:

# Kiểm tra xem phần tử có được chọn không
checkbox = xml_dump.find_element_by_attribute("class", "android.widget.CheckBox")
if checkbox:
    is_checked = xml_dump.is_element_checked(checkbox)
    print(f"Checkbox đã được chọn: {is_checked}")

# Kiểm tra xem phần tử có được bật không
button = xml_dump.find_element_by_text("Submit")
if button:
    is_enabled = xml_dump.is_element_enabled(button)
    print(f"Nút được bật: {is_enabled}")

# Kiểm tra xem phần tử có được focus không
input_field = xml_dump.find_element_by_attribute("focused", "true")
if input_field:
    print("Đã tìm thấy phần tử đang được focus")

Tối ưu hiệu suất

Truy xuất cấu trúc XML của UI có thể tốn thời gian, đặc biệt là khi bạn cần thực hiện nhiều lần. OIADB cung cấp XML Server để cải thiện hiệu suất:

from oiadb.commands import xml_dump

# Khởi động XML Server
server, server_thread = xml_dump.start_xml_server(port=8000)

# Thiết lập port forwarding
xml_dump.setup_port_forwarding(device_port=8000, host_port=8000)

# Sử dụng XML Server để lấy cấu trúc UI (nhanh hơn)
for i in range(10):
    xml_tree = xml_dump.get_ui_xml_from_server()
    # Thực hiện các thao tác với xml_tree
    
    # Giả lập độ trễ giữa các lần kiểm tra
    import time
    time.sleep(1)

# Dừng XML Server khi hoàn tất
xml_dump.stop_xml_server(server)

Mẹo tối ưu hiệu suất

  • Sử dụng XML Server khi cần truy xuất cấu trúc UI nhiều lần
  • Lưu trữ và tái sử dụng cấu trúc XML khi có thể
  • Sử dụng các phương thức tìm kiếm cụ thể (find_element_by_attribute) thay vì duyệt qua toàn bộ cấu trúc
  • Kết hợp XML Dump với nhận diện hình ảnh để xử lý các trường hợp phức tạp
